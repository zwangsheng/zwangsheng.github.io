Spring IOC

控制反转的思想，将程序原本直接创建对象的权利转交给Spring框架

IOC容器来为我们注入我们需要的对象

Spring AOP

面向切面编程，将那些业务无关，但是被业务需要的代码封装起来，降低耦合

AOP基于动态代理，实现了接口就用JDK动态代理，未实现的就用CGLIB来代理

SpringBean 的作用域

Singleton

单例，默认单例

Prototype

每次请求这个bean都会产生一个新的bean

Request

每一次Http请求就会产生新的bean，仅在http request有效

Session

每一次Http请求就会产生新的bean，仅在http session有效

@Bean 和 @Component的区别

  1. 作用对象不同
    @Component作用于对象，@Bean作用于方法
  2. 使用场景
    @Component一般是扫描的时候标注进行自动装配，@Bean声明这个方法产生bean
