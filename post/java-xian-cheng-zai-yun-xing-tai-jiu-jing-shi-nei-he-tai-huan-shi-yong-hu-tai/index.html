Java线程在运行态究竟是内核态还是用户态？

2020-12-09 2 min read # Java # 多线程

🈚️解，一个软件层面的线程的用户态和内核态是不能确定的
解析一下内核态和用户态
首先，内核态和用户态只是CPU执行权限
用户态切换到内核态的方式

  1. 系统调用
  2. 操作系统层面的异常
  3. 外设的中断
    目前了解的就是第一部分，java调用方法，唤起native方法，对应到OS的函数，是线程进入内核态。
    ex.线程的park方法，对应OS的某个函数，切换到内核态
    内核态就是进程在内核空间运行的状态
    用户态就是进程在用户空间运行的状态

CPU的上下文

CPU的上下文可以狭义的理解为CPU的寄存器和程序计数器
（可以在JVM中找到对应的模拟，所以JVM可以看作虚拟的小型OS）

CPU上下文的类型

进程上下文的切换

这与我们说的进程上下文切换不同，系统调用的进程上下文切换针对的是同一个进程
可以理解为这是一种权限切换

  1. 保存CPU寄存器中的用户态指令位
  2. CPU寄存器更新为内核态的指令位
  3. 跳转到内核态运行任务
  4. 系统调度结束之后，恢复原先保存的用户态，切换回用户态，继续运行

“进程上下文切换”

我们所谈到的进程上下文切换是多个进程进行切换的情况，它比上述的系统调用的上下文切换多一个步骤即为保存进程内部的虚拟资源
发生“进程上下文切换”的情况

  1. CPU时间轮转
  2. CPU资源（不单单是CPU时间）不足，进程被挂起
  3. 进程主动挂起自己
  4. 高优先级进程运行

线程上下文的切换

分场景
如果两个线程属于不同的进程，那么切换就像进程上下文切换一样
如果两个线程属于同一个进程，那么切换的时候，虚拟内存不动，只需要切换线程私有的资源即可
